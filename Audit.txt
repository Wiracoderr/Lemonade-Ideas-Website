# 🚨 FINDINGS CLASSIFICATION

### 1. Hreflang and Sitemap Translation Missing (CRAWLABILITY & INDEXATION)
* Issue: `sitemap.ts` generates separate URLs for `/en/` and `/es/` but completely lacks explicit Next.js `alternates.languages` configurations (hreflang nodes).
* Evidence: Analyzed `src/app/sitemap.ts` lines 18-35. It maps locales blindly without attaching `alternates` metadata.
* Severity: Critical (Causes search engines to index pages as duplicates rather than linguistic alternates).
* Confidence: High.
* Why It Matters: Google will penalize the Spanish pages for cannibalizing the English pages because it does not have the XML structural signal telling it they are identical pages targeting different demographics.
* Score Impact: -30 (Crawlability)
* Recommendation: Refactor `sitemap.ts` to utilize the Next.js `alternates.languages` syntax mapping `en` and `es` to their specific URLs per object.

### 2. Missing Global Canonical Tags on Core Pages (ON-PAGE OPTIMIZATION)
* Issue: The main landing pages (e.g. Home, `/contact`, `/pricing`) lack explicit canonical URL definitions in their Next.js Metadata objects.
* Evidence: Analyzed `src/app/[locale]/page.tsx` line 18. No `alternates: { canonical: ... }` rule is declared. (Whereas the new blogs DO have this).
* Severity: Medium.
* Confidence: High.
* Why It Matters: Without `canonical` tags, parameter-driven URL variations (e.g. `?utm_source=facebook` or direct IP loads) will be indexed as separate pages.
* Score Impact: -5 (On-Page Optimization)
* Recommendation: Inject dynamic `alternates: { canonical: ... }` into all static page `metadata` objects.

### 3. Lack of Author Schema / Structured Data (CONTENT QUALITY & E-E-A-T)
* Issue: The newly generated 8-figure construction blogs are highly advanced but lack `JSON-LD Schema Markup` identifying the author or organization.
* Evidence: Reviewed Blog 1-5 `page.tsx` creations. No structured data `<script type="application/ld+json">` is injected.
* Severity: Medium.
* Confidence: High.
* Why It Matters: Google's E-E-A-T update demands cryptographic proof of who wrote an article. Without JSON-LD, Google struggles to attribute the "Sentry Tone" expertise to a real contracting entity.
* Score Impact: -5 (Content Quality)
* Recommendation: Build a `BlogSchema` React component that automatically accepts the blog slug and injects Google-compliant Article JSON-LD on render.

### 4. Lingering <a> Tag Navigation vs <Link> (TECHNICAL FOUNDATIONS)
* Issue: Over 2,500 ESLint errors plague the root directory, primarily citing `no-html-link-for-pages` warnings where standard `<a>` tags are used instead of Next.js `<Link>`.
* Evidence: Detected in prior compilation runs.
* Severity: Low (Cosmetic to SEO, matters for User Experience & INP).
* Confidence: Medium.
* Why It Matters: Using standard `<a>` tags breaks Single Page Application (SPA) client-side routing, forcing a full server reload on every click. This tanks the Interaction to Next Paint (INP) metric.
* Score Impact: -5 (Technical Foundations)
* Recommendation: Run an automated global replace of `<a>` internal links with `next/link`.
